# About
Meant as a portfolio for a quick glance over my programming skills and mastery.

## Portfolio
### Files
Anytime a coding challenge has been complete, a commit is being made which acts as a record. It is only done when I fully grasp and understood a topic and I am capable to write the program again without any help.

### Commitment
As mentioned before, I spend each day focusing on a topic. I am taking my time to fully understand and grasp what I am learning, for example: What are lists? How can you work with them? How do you pass an argument? In the evening, before calling it a day, I complete coding challenges, including the ones from previous days which I dubbed the Rite of Coding.

### Rite of Coding
The Rite of Coding is where I finish my day completing coding challenges. These range from the latest topic I completed as well as having randomly choosen one that I already completed. This helps me to ensure to see where my knowledge is firm and where I need to work on again. Only when I completed a challenge three times without any assistance, the file gets committed to this repository as part of the portfolio.

#### Conditions
The conditions are simple: No textbook, no internet, no A.I. Only pure skill, knowledge and the Geany IDE (I love to work with Geany)

## Key Audience Target
The main audience are for employers who would like to get a direct, raw and unfiltered overview of my hard skills. Nothing less.
